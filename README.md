### 📂 Phase 1: Go & Linux Foundations

#### 🔹 Day 1: The "Why" of Go & The Classic Hello World
* **Core Concepts Mastered:**
    * **Lightweight Compilation:** Understood why Go is the king of cloud-native infrastructure. Because it compiles directly to native machine code with no heavy VM or interpreter layer, it achieves tiny disk sizes and sub-millisecond startup times—critical for rapid container autoscaling.
    * **Memory Efficiency:** Explored how Go minimizes CPU/memory overhead compared to interpreted languages.
    * **Concurrency Model:** Introduced to the theory of Goroutines and how Go efficiently handles heavy workloads at the system level.
    * **Type System:** Learned the benefits of strict static typing and structural compilation.
* **Code Implemented:**
    * Initialized `package main` and executed the standard entry point (`func main()`).
    * Imported and utilized the `fmt` standard library package.
    * Explored explicit variable declaration (`var`) and dynamic type inference (`:=`).
